# anticrab.nvim — Cheatsheet

Лидер — `Space`. Через `dual()` почти все маппинги работают и в русской раскладке (Ф = Space, Ы = S и т.д.), переключаться не обязательно. `jk` в insert-режиме = `<Esc>`. `;` = `:`.

>  **Подсказка по биндингам в реальном времени:** жмёшь `<Space>` и через 0.3 сек выскочит окошко which-key со всеми возможными продолжениями. `<leader>?` — посмотреть все маппинги, доступные в текущем буфере.

## Vim — базовое редактирование (для тех кто только начал)

### Режимы

| Из → в | Клавиша |
|---|---|
| Normal → Insert | `i` (перед курсором), `a` (после), `I` (начало строки), `A` (конец строки), `o` (новая строка ниже), `O` (новая строка выше) |
| Insert → Normal | `<Esc>` или `jk` |
| Normal → Visual | `v` (по символам), `V` (по строкам), `<C-v>` (блочный) |
| Normal → Command | `:` или `;` (в этом конфиге `;` = `:`) |
| Замена символов | `r<char>` (один символ), `R` (replace mode) |

### Перемещение курсора

| Клавиша | Действие |
|---|---|
| `h` `j` `k` `l` | Влево / вниз / вверх / вправо |
| `w` / `b` | Следующее / предыдущее слово |
| `e` | Конец слова |
| `0` / `^` | Начало строки / первый непробельный символ |
| `$` | Конец строки |
| `gg` / `G` | Начало / конец файла |
| `{` / `}` | Предыдущий / следующий пустой абзац |
| `%` | Парная скобка `({[` |
| `f<char>` / `F<char>` | Прыжок к символу вправо / влево на текущей строке |
| `t<char>` / `T<char>` | То же, но **до** символа |
| `;` / `,` | Повторить последний `f/t` вперёд / назад (но `;` у нас перенастроена в `:`, используй `,`) |
| `*` / `#` | Поиск слова под курсором вперёд / назад |
| `<C-d>` / `<C-u>` | Полстраницы вниз / вверх |
| `<C-o>` / `<C-i>` | Назад / вперёд по jump-list (как back/forward в браузере) |
| `<leader>ss` | **Flash jump** — наберёшь 2 буквы, прыгнешь на метку |

### Удаление / изменение / копирование (operator + motion)

Базовая идея: `<operator><motion-or-textobject>`. Например `dw` = delete word, `c$` = change to end of line.

| Оператор | Действие |
|---|---|
| `d` | Удалить (cut в регистр) |
| `c` | Удалить и войти в insert (change) |
| `y` | Скопировать (yank) |
| `p` / `P` | Вставить после / до курсора |
| `x` | Удалить символ |
| `u` / `<C-r>` | Undo / Redo |
| `>>` / `<<` | Сдвинуть строку вправо / влево |
| `==` | Авто-отступ строки |
| `.` | Повторить последнее изменение |

Удвоенный оператор работает на всю строку: `dd` удалить строку, `yy` скопировать строку, `cc` изменить строку.

### Text objects (то, на чём работают операторы)

Шаблон: `<operator>{i|a}<object>`. `i` = inner (без обёртки), `a` = around (с обёрткой).

| Объект | Что |
|---|---|
| `iw` / `aw` | Слово |
| `is` / `as` | Предложение |
| `ip` / `ap` | Параграф |
| `i"` `i'` `` i` `` | Внутри кавычек |
| `i(` `i[` `i{` `i<` | Внутри скобок |
| `it` / `at` | HTML/XML тег |

Примеры: `ciw` — заменить слово, `da"` — удалить вместе с кавычками, `vi{` — выделить внутри `{}`, `yi(` — скопировать содержимое скобок.

### Поиск и замена

| Клавиша / команда | Действие |
|---|---|
| `/foo` | Поиск `foo` вперёд |
| `?foo` | Поиск назад |
| `n` / `N` | Следующее / предыдущее совпадение |
| `:%s/foo/bar/g` | Заменить `foo` на `bar` во всём файле |
| `:%s/foo/bar/gc` | То же с подтверждением каждой замены |
| `:s/foo/bar/g` | Только в текущей строке |
| `:noh` | Убрать подсветку поиска |

### Marks и регистры

| Команда | Действие |
|---|---|
| `m<a-z>` | Поставить метку |
| `'<a-z>` / `` `<a-z> `` | Прыжок к метке (строка / точная позиция) |
| `"<a-z>y` / `"<a-z>p` | Скопировать в / вставить из именованного регистра |
| `"+y` / `"+p` | Системный буфер обмена |
| `:reg` | Посмотреть содержимое всех регистров |

## Git — gitsigns

Подсветка добавленных/изменённых/удалённых строк в gutter включена. Маппингов под `<leader>` сейчас нет, всё доступно через команды:

| Команда | Действие |
|---|---|
| `:Gitsigns preview_hunk` | Превью hunk во всплывашке |
| `:Gitsigns next_hunk` / `prev_hunk` | Прыжок между hunk'ами |
| `:Gitsigns blame_line` | Кто менял строку |
| `:Gitsigns toggle_current_line_blame` | Inline-blame постоянно |
| `:Gitsigns stage_hunk` / `undo_stage_hunk` | Stage hunk |
| `:Gitsigns reset_hunk` | Откатить hunk |

## Терминал

**Quick float — для коротких ad-hoc команд:**

| Клавиша | Действие |
|---|---|
| `<leader>/` | Toggle плавающего терминала (одна общая сессия) |

**Numbered терминалы — обычные buffer-вкладки (видны сверху в barbar):**

| Клавиша | Действие |
|---|---|
| `<leader>t1` … `<leader>t9` | Открыть/перейти на terminal-buffer #N |

Это **обычные буферы**, такие же как открытые файлы — появляются сверху во вкладках barbar (`term-1`, `term-2`, …). Переключаются как любые буферы (`<leader>tn` / `<leader>tp`), закрываются `<leader>tx` (terminal-процесс убивается). Пока буфер жив — в нём сохраняется shell со всей историей; нажал `<leader>t1` снова — попадёшь обратно туда же.

**Внутри терминала:**

| Клавиша | Действие |
|---|---|
| `jk` | Выйти в normal-mode (терминал не закрывается) |
| `<C-\><C-n>` | То же, канонический способ |
| `<C-h/j/k/l>` | Перейти в соседнее окно |

В terminal-mode `<Esc>` оставлен TUI-приложениям (lazygit, htop, fzf), чтобы они работали корректно.

**Если нужен сплит-терминал на момент:**

```
:ToggleTerm direction=horizontal size=20      " снизу высотой 20
:ToggleTerm direction=vertical size=80        " справа шириной 80
```

## Сессии (auto-session)

Автосохранение и автовосстановление сессий по `cwd`. Команды: `:SessionSave`, `:SessionRestore`, `:SessionDelete`. Перед сохранением закрываются скрытые табы и буферы.

## Прочее (без биндингов)

- **bigfile.nvim** — отключает тяжёлые фичи в файлах >2 MiB.
- **indent-blankline** — направляющие отступа `|`.
- **barbecue** — breadcrumbs по LSP в верхней строке окна.
- **lualine** — status line, тема `catppuccin-nvim`.
- **conform.nvim** — форматирование на сохранении (форматтеры пока не настроены, добавим под Python/C++).
- **todo-comments** — подсветка `TODO/FIXME/HACK` (`:TodoTelescope`, `:Trouble todo`).
- **treesitter** — подсветка для `lua/python` + автоустановка для новых языков.

## Подсказки по биндингам (which-key)

| Клавиша | Действие |
|---|---|
