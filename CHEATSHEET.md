# anticrab.nvim — Cheatsheet

Leader key: `Space`. Almost every `<leader>X` mapping has a Russian-keyboard alias generated by the `dual()` helper (Ф = Space, Ы = S, etc.), so you don't need to switch layout. `jk` in insert mode = `<Esc>`. `;` = `:`.

>  **Live keybinding hints:** press `<Space>` and after 0.3s the which-key popup shows every available continuation. `<leader>?` lists every mapping active in the current buffer.

## Vim — basic editing (for newcomers)

### Modes

| From → To | Keys |
|---|---|
| Normal → Insert | `i` (before cursor), `a` (after), `I` (line start), `A` (line end), `o` (new line below), `O` (new line above) |
| Insert → Normal | `<Esc>` or `jk` |
| Normal → Visual | `v` (charwise), `V` (linewise), `<C-v>` (blockwise) |
| Normal → Command | `:` or `;` (this config remaps `;` → `:`) |
| Replace single char | `r<char>` (one char), `R` (replace mode) |

### Cursor movement

| Key | Action |
|---|---|
| `h` `j` `k` `l` | Left / down / up / right |
| `w` / `b` | Next / previous word |
| `e` | End of word |
| `0` / `^` | Line start / first non-blank |
| `$` | End of line |
| `gg` / `G` | Top / bottom of file |
| `{` / `}` | Previous / next blank-line paragraph |
| `%` | Matching bracket `({[` |
| `f<char>` / `F<char>` | Jump to char on current line, forward / backward |
| `t<char>` / `T<char>` | Same but stops **before** the char |
| `,` | Repeat last `f/t` (note: `;` is remapped to `:`, so use `,` only) |
| `*` / `#` | Search word under cursor forward / backward |
| `<C-d>` / `<C-u>` | Half page down / up |
| `<C-o>` / `<C-i>` | Jump-list back / forward (browser-style) |
| `<leader>ss` | **Flash jump** — type 2 chars, pick a label |

### Delete / change / yank (operator + motion)

Pattern: `<operator><motion-or-textobject>`. Examples: `dw` = delete word, `c$` = change to end of line.

| Operator | Action |
|---|---|
| `d` | Delete (cuts into register) |
| `c` | Delete and enter insert mode (change) |
| `y` | Yank (copy) |
| `p` / `P` | Paste after / before cursor |
| `x` | Delete character |
| `u` / `<C-r>` | Undo / redo |
| `>>` / `<<` | Indent line right / left |
| `==` | Auto-indent line |
| `.` | Repeat last edit |

Doubling the operator acts on the whole line: `dd` delete line, `yy` yank line, `cc` change line.

### Text objects (operator targets)

Pattern: `<operator>{i|a}<object>`. `i` = inner (without delimiters), `a` = around (with delimiters).

| Object | What |
|---|---|
| `iw` / `aw` | Word |
| `is` / `as` | Sentence |
| `ip` / `ap` | Paragraph |
| `i"` `i'` `` i` `` | Inside quotes |
| `i(` `i[` `i{` `i<` | Inside brackets |
| `it` / `at` | HTML/XML tag |

Examples: `ciw` — replace word, `da"` — delete with quotes, `vi{` — visual select inside `{}`, `yi(` — yank inside parens.

## Docker projects (`<leader>D*`)

Active only when a `.nvim-docker.lua` marker is found at the project root. Marker schema and full description live in the README's "Docker-based projects" section.

All project-defined `<leader>D<key>` commands (and `<leader>Ds`) share **one persistent in-container bash** in its own tab — output history accumulates, scrollable. `q` in normal-mode closes the tab; the shell keeps running; reopen via `<leader>Ds`.

**Built-in (always registered, keys reserved):**

| Key | Action |
|---|---|
| `<leader>Ds` | Focus / open the persistent shell tab |
| `<leader>Dr` | Restart wrapped LSPs — run after every rebuild (inotify across bind-mounts is unreliable) |
| `<leader>Di` | Print marker info (root, mappings, container status, wrapped servers, project commands, shell state) |
| `<leader>Dl` | Tail container logs (separate tab) |

**Project-defined** — each entry of `commands = { ... }` in the marker becomes `<leader>D<key>`. Example:

```lua
commands = {
  { key = "b", desc = "Build",         cmd = "make" },
  { key = "B", desc = "Build target",  select_from = "ls -1 cmd/", cmd = "go build ./cmd/{pkg}" },
  { key = "t", desc = "Test",          cmd = "go test ./..." },
}
```

If `select_from` is set, `vim.ui.select` prompts with one option per line of that command's output (run inside the container); the chosen value replaces `{pkg}` in `cmd`.

`:checkhealth core.docker-project` — full diagnostic: marker / container / clangd reach / compile_commands / latency / mapping sanity.

**Trust prompt:** `vim.secure.read` asks once on first encounter / content change. After choosing `view`, switch to the marker buffer and run `:trust` (it acts on the **current** buffer — don't run it while you're on a source file). Choice persisted in `~/.local/state/nvim/trust`.

**Container must ship these tools** (add to your project's Dockerfile):

| Tool | Used by | Install |
|---|---|---|
| `clangd` | clangd LSP (C/C++) | `apt install clangd-15` + `update-alternatives` |
| `pyright-langserver` | pyright LSP | `pip3 install pyright` |
| `jq` | `<leader>Db` compile_commands aggregator | `apt install jq` |
| `ament_flake8`, `ament_pep257` | nvim-lint (ROS2 Python) | `apt install ros-<distro>-ament-flake8 ros-<distro>-ament-pep257` |
| `clang-format`, `clang-tidy` | conform / nvim-lint | `apt install clang-format clang-tidy` |

**Cold container:** if container is down, no wrapped LSP starts and a single notify fires. `<leader>Ds` to enter shell, `make up` (or your equivalent), then `<leader>Dr`.

**Known limitations:**
- Jump-to-def into `/opt/ros/humble/...` shows a container-only path that doesn't exist on host.
- Rebuilds need explicit `<leader>Dr` (inotify across bind-mounts isn't reliable on Linux).
- First `docker compose exec` per session adds ~200–500 ms (compose resolve).
